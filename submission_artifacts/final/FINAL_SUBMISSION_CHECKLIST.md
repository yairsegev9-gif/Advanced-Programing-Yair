# Final Submission Checklist

## What To Push To Git

Push the complete integrated project repository on branch `codex-complete-project` or merge it only if your course workflow explicitly asks for main. The Git repository should include:

- `src/` with all Java sources
- `config_files/simple.conf`
- `html_files/`
- `README.md`
- `REQUIREMENTS_CHECKLIST.md`
- `FINAL_REPORT.md`
- `REVIEW_GUIDE.md`
- `docs/javadoc/`
- `submission_artifacts/video/` materials if you want them versioned

Do not push generated `.class` files, `.idea/`, or `out/`.

## What To Place In link.txt

Edit `submission_artifacts/final/link.txt` and fill in:

- Git repository URL accessible to the grader
- Submitter name, ID, and email
- Second submitter details, or remove the second block if submitting alone

## Where To Submit link.txt

Submit `link.txt` to the dedicated final Exercise 6 module submission box. The final integrated project is submitted by repository link, not through the Exercises 1-5 automatic testing system.

## Accessibility Verification

Before submitting:

1. Open the repository URL in a private/incognito browser window or institutional account.
2. Confirm the grader can access the repository without local machine paths.
3. Confirm README includes the demo video link after you upload the video.
4. Confirm `docs/javadoc/index.html` exists in the repository.
5. Confirm the repository branch or default branch contains the final commits.

## Exercises 1-5

Exercises 1-5 have separate ZIPs under `submission_artifacts/`. Submit those ZIPs to the relevant course testing/module system entries, not to the final Exercise 6 link box.
